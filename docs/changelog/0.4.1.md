# [0.4.1-SNAPSHOT] — Unreleased

### Added

**Part A — Critical:**
- **True async driver usage** — `executeSuspend`/`executeSuspendAll`/`prepareSuspend` extensions on `CqlSession`; all suspend write paths in `BatchEngine` and, as of this release, all suspend read paths in `QueryExecutor` (`findByIdSuspend`, `findAllSuspend`, `findPageSuspend`, `existsSuspend`, `rawSuspend`, `rawQuerySuspend`) use true async via `executeAsync().await()` — no coroutine dispatcher thread is ever blocked on a network round-trip (see [ISS-014](../issues/ISS-014-blocking-query-executor.md))
- **UNSET vs NULL** — `KandraUnset` sentinel; `encode()` returns `KandraUnset` for null nullable fields → `stmt.unset(idx)` in `insertPrimary` (no tombstone); `saveWithNulls()` for intentional tombstone writes
- **Optimistic locking** — `@Version` on `Long`/`Instant` fields; `save()` sets version to `1L`/`Instant.now()`; `update()` uses LWT `IF version = ?`, throws `KandraOptimisticLockException` on conflict; `updateForce()` bypasses the check
- **Tombstone awareness** — `@SoftDelete(ttlSeconds)` → `delete()` sets TTL instead of DELETE; `@SoftDelete(markerProperty = "...")` adds an opt-in permanent marker column enabling `findActive()` (see [ISS-007](../issues/ISS-007-find-active-soft-delete.md)); `@ScyllaTable(gcGraceSeconds=N)` → DDL includes `WITH gc_grace_seconds = N`; `alterTableAddColumn()` added to `DdlGenerator`
- **Batch size guard** — `batchWarnThresholdKb`, `batchMaxChunkSize`, `batchAutoChunk` config; `saveAll()` auto-chunks large batches
- **`@Sensitive` log masking** — `ColumnSchema.isSensitive`; `KandraEntityLogger.safeToString()` masks sensitive fields with `***`

**Part B — Should-have:**
- **Health check** — `GET /kandra/health` route (JSON `{"status":"UP/DOWN"}`); enabled via `KandraConfig.healthCheck = true`
- **Graceful shutdown** — `ShutdownConfig(drainTimeoutMs, graceful)`; `KandraRuntime.isShuttingDown` + `inFlightCount`; `ApplicationStopping` event drains in-flight queries before close
- **Validation hook** — `KandraValidator<T>`, `KandraValidationError`, `KandraValidationException`; `validate<User> {}` in plugin config; runs before every save/update
- **AUTO_MIGRATE** — new `SchemaMode.AUTO_MIGRATE` → `CREATE TABLE IF NOT EXISTS` + diff against `system_schema.columns` + `ALTER TABLE ADD` for new columns; logs WARN for unmapped Scylla columns
- **Micrometer metrics** — `MetricsConfig` in `KandraConfig` (enabled=false by default, meterRegistry, tagTableName, tagOperationType); deps as `compileOnly`
- **CQL injection guard** — `raw()` logs WARN when called with no params but CQL contains string literals; `KandraRawQuery` builder for safe parameterised queries; `rawQuery()` on both repos
- **Schema migration runner** — `kandra-migrate` module with `KandraMigration`, `KandraMigrationRunner` (versioned, checksum-validated, claim-locked — see [ISS-018](../issues/ISS-018-migration-no-locking.md)), `MigrationHistory`
- **Query caching** — `@CacheResult(ttlSeconds, maxSize)` → Caffeine-backed `findById` cache; invalidated on save/update/delete; `compileOnly` dep (disabled gracefully if not on classpath)

**Part C — Publishing & ecosystem:**
- `gradle/libs.versions.toml` — version catalog wired into all module build files
- `.github/workflows/ci.yml` — `docs` job runs `dokkaHtmlMultiModule` on tag push, deploys to GitHub Pages
- **`kandra-jakarta`** module — `JakartaKandraValidator<T>` adapts `jakarta.validation.Validator` into `KandraValidator<T>`; `validateJakarta<T>()` extension on `KandraConfig`; `jakarta.validation-api` is `compileOnly`, bring your own provider (e.g. Hibernate Validator) (see [ISS-011](../issues/ISS-011-jakarta-bean-validation.md))
- **Real-cluster integration tests** — `kandra-test/src/test/.../KandraIntegrationTest.kt` and re-enabled `kandra-ktor` plugin tests, run via `KandraTestcontainers` against real Cassandra (see [ISS-013](../issues/ISS-013-no-integration-tests.md))

### Fixed

Found during a pre-real-database-testing audit; see the linked issue for each for full detail:

- [ISS-014](../issues/ISS-014-blocking-query-executor.md) — `QueryExecutor` blocked the calling coroutine dispatcher on every suspend read
- [ISS-015](../issues/ISS-015-delete-by-id-bypasses-soft-delete.md) — `KandraRepository.deleteById` bypassed `@SoftDelete`, unlike every other delete path
- [ISS-017](../issues/ISS-017-migration-checksum-not-body-based.md) — migration checksum didn't actually hash the migration body
- [ISS-018](../issues/ISS-018-migration-no-locking.md) — concurrent migration runners could double-apply a migration
- [ISS-019](../issues/ISS-019-collection-counter-consistency.md) — collection/counter statements ignored configured consistency levels
- [ISS-021](../issues/ISS-021-allow-filtering-error-message.md) — an error message referenced a non-existent `allowFiltering()` DSL method

### Changed
- `StatementBuilder.appendToCollection`, `removeFromCollection`, `counterUpdate` gained an optional `consistency: KandraConsistency?` parameter, threaded through `KandraRepository`/`KandraSuspendRepository`'s `append`/`remove`/`put`/`increment`/`decrement`
