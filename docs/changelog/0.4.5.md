# [0.4.5] — Unreleased

### Changed

- **Breaking:** removed the `+` unary-plus predicate registration from the query DSL
  (`repo.find { +UserTable.email.eq("x") }` -> `repo.find { UserTable.email eq "x" }`). Flagged
  independently in two rounds of external code review as the library's biggest DSL footgun: forgetting
  the `+` on one predicate out of several compiled cleanly and silently dropped that predicate from the
  query, with no way to detect it at runtime (the discarded `KandraPredicate` never reached
  `QueryContext`). `eq`/`gt`/`gte`/`lt`/`lte`/`isIn` are now `QueryContext` member extensions instead of
  `KandraColumnRef` methods — the comparison itself performs the registration, so there is no
  intermediate value left to forget. See
  [GitHub issue #4](https://github.com/Dev-Pasaka/kandra/issues/4) for the full writeup.

  Migration: replace every `+Table.col.op(value)` with `Table.col op value` (drop the leading `+`,
  everything else about the call is unchanged — same operator names, same infix-call shape).
